Preguntas del Laboratorio 1

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de computadoras interconectadas, mientras que la World Wide Web (WWW) es un sistema de información basado en hipertexto que opera sobre Internet.

¿Cuáles son las partes de una URL?

  • Protocolo: Define cómo se accede al recurso (ej. http://, https://).
  • Dominio: Dirección del servidor (ej. www.ejemplo.com).
  • Ruta: Ubicación del recurso en el servidor (ej. /pagina.html).
  • Parámetros: Información adicional enviada en la URL (ej. ?id=123).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita datos sin modificarlos.
  • HEAD: Igual que GET, pero solo obtiene encabezados.
  • POST: Envía datos al servidor.
  • PUT: Crea o actualiza un recurso.
  • PATCH: Modifica parcialmente un recurso.
  • DELETE: Elimina un recurso.

¿Cuál es tu recomendación sobre el uso de !important en CSS?

Debe usarse con moderación, ya que puede dificultar la gestión del código al anular la especificidad de los estilos.

¿Por qué debe escogerse con cuidado una imagen de fondo en una página HTML?

  • Puede afectar la legibilidad del contenido.
  • Un archivo grande puede ralentizar la carga del sitio.
  • Si no está bien ajustada (background-size), puede cortarse o repetirse de forma no deseada.

Preguntas del Laboratorio 2

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

El uso de !important en CSS debe ser limitado y justificado, ya que anula la especificidad normal de los estilos y puede dificultar el mantenimiento del código. Se recomienda solo en casos específicos, como cuando se necesita sobrescribir estilos de terceros o corregir errores urgentes. Lo ideal es estructurar bien las reglas CSS para evitar depender de !important.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

  • Legibilidad: No debe afectar la visibilidad del texto ni generar un contraste deficiente.
  • Rendimiento: Imágenes pesadas pueden ralentizar la carga de la página.
  • Compatibilidad: Debe ajustarse correctamente en diferentes dispositivos y tamaños de pantalla (usar background-size: cover;).
  • Accesibilidad: No debe ser el único medio para transmitir información importante.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de una propiedad de estilo entre %, px y pt?

  • % (porcentajes): Se recomienda para diseños responsivos, ya que permite adaptar el tamaño en función del contenedor.
  • px (píxeles): Útil cuando se requiere un tamaño fijo, pero puede no adaptarse bien en pantallas de diferentes resoluciones.
  • pt (puntos): Más usado en impresión, no recomendado para pantallas porque no escala bien con diferentes resoluciones.

Lo ideal es usar unidades relativas como %, em, rem o vw/vh para garantizar flexibilidad en el diseño.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

La versión minimizada de un CSS elimina espacios, comentarios y caracteres innecesarios, reduciendo el tamaño del archivo. Esto mejora el rendimiento porque:

  • Carga más rápida: Reduce el tiempo de descarga y renderizado.
  • Menos uso de ancho de banda: Beneficia a usuarios con conexiones lentas.
  • Mejor caché y rendimiento en servidores: Archivos más pequeños son más eficientes en almacenamiento y distribución.

Siempre es recomendable usar CSS minimizado en producción y mantener una versión legible para desarrollo.
